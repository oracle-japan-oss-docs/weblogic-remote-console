既知の問題

このページには、WebLogic Remote Consoleの制限と既知の問題(および使用可能な場合はその回避策)が一覧表示されます。

制限

現在のリリースのWebLogic Remote Consoleでは、次のことはできません:

  • ユーザー、グループ、ロール、ポリシー、資格証明マップなど、WebLogic構成の外部にあるWebLogic Serverセキュリティ・プロバイダのセキュリティ・データを管理します。

  • WebLogic Remote Consoleを使用してWebLogic Serverを構成中にWLSTスクリプトを記録します。

  • JNDIツリー・ビューア

  • (macOS)複数のWebLogic Remote Consoleアプリケーションを一度に実行します。 これには、WebLogic Kubernetes Toolkit UI内で実行されているWebLogic Remote Consoleインスタンスは含まれません - WebLogic Remote ConsoleおよびWKT UIを同時に実行できます。

webブラウザの最小ビューポート・サイズ

「問題」 WebLogic Remote Consoleのフォント・サイズは、Oracle JET Redwoodスタイルによって制御されます。 これらは、WebLogic Server管理コンソールで使用されるフィールドよりも大きく、入力フィールドは高さと幅が少し大きくなります。 その結果、webブラウザのビューポート内の全体の表示領域(すべてのWebコンテンツが表示される部分)は、WebLogic Remote Consoleを使用して楽しい経験を得るために最小サイズである必要があります。

推奨される最小webブラウザ・ビューポート・サイズは1592 x 900ピクセルです。 webブラウザのサイズを1592 x 900ピクセルよりも小さくすると、マウスが特定の領域にマウスを移動したときにスクロール・バーが表示されます。 ただし、一部のフォーム・データは、webブラウザのビューポートの高さが900ピクセル以上になるまで表示されない場合があります。

「回避策」ブラウザ・ビューポートが推奨される最小値に設定されていることを確認します。 これらの値が使用できない場合は、ズーム設定を80%に調整することもできます。

制限されたMBeanプロパティのサポート

WebLogic Remote Consoleには、WebLogic Server管理コンソールでサポートされるMBeanプロパティの大部分が含まれます。 ただし、WebLogic REST APIの制限により、一部が欠落しており、一部は非推奨です。 WebLogic Remote Consoleで使用できないMBeanプロパティが必要な場合、お知らせください。

セキュリティ・プロバイダなどのコレクションの順序を変更できません

「問題」セキュリティ・プロバイダのタイプ(認証プロバイダ、ロール・マッピング・プロバイダ、認可プロバイダなど)ごとに個別のリストがあります。 リストに複数のセキュリティ・プロバイダがある場合、WebLogic Serverは、リストに表示される順序でセキュリティ・プロバイダを起動します。 オーダーが重要になる場合があります(たとえば、サーバーが低速のリモート・プロバイダの前にローカル高速認証プロバイダを試すようにする場合)。

現在、WebLogic Remote Consoleでは、リストを明示的に並べ替えることはできません。

「回避策」プロバイダを並べ替えるには、プロバイダを削除して再作成できます。 たとえば、認証プロバイダのリストにAがあり、Bに切り替える場合はAを削除して再作成します。 新しいプロバイダは常にリストの末尾に追加されます。
