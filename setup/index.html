開始

WebLogic Remote Consoleは、デスクトップ・アプリケーションまたはブラウザ・アプリケーションのいずれかとしてインストールおよび実行できます。 どちらのオプションも有効な選択肢ですが、多少異なる機能があります。

可能な限り、ブラウザ・アプリケーションよりもデスクトップ・アプリケーションを使用することをお勧めします。 WDTモデル・ファイルの編集などの特定の機能は、デスクトップ・アプリケーションでより効果的に機能します。

システム要件の確認

WebLogic Remote Consoleは、次のプラットフォームをサポートしています:

  • Linux (64ビットのみ)
    • Ubuntu 14.04以降
    • Fedora 24以降
      • Oracle Linux 8以降
    • Debian 8以降
  • macOS
    • macOS 10.11 (El Capitan)以上(64ビットIntel)
  • Windows (64ビットのみ)
    • Windows 7以降

また、WebLogic Remote Console 「ブラウザ・アプリケーション」では、最新のインターネット・ブラウザを使用する必要があります:

  • Apple Safari
  • Google Chrome
  • Microsoft Edge
  • Mozilla Firefox

WebLogic Remote Console 「デスクトップ・アプリケーション」は、Electronフレームワークに基づいています。

WebLogic Remote Consoleのインストール

デスクトップ・アプリケーション

  1. コンピュータから以前のバージョンのWebLogic Remote Consoleをアンインストールします。
  2. WebLogic Remote Consoleをダウンロードします。 https://github.com/oracle/weblogic-remote-console/releasesに移動して、オペレーティング・システムに適用可能なインストーラをダウンロードします。
  3. インストーラを実行します。
  4. オプション: WebLogic Remote Console拡張機能をWebLogic Serverドメインにインストールします。 WebLogic Remote Console拡張では、WebLogic Remote Consoleを使用するときに追加機能が追加されます。
    1. ドメイン・ホームの下にmanagement-services-extディレクトリを作成します。
    2. https://github.com/oracle/weblogic-remote-console/releasesからconsole-rest-ext-2.0.warをダウンロードし、前のステップで作成したmanagement-services-extディレクトリ内に保存します。 以前のバージョンの拡張機能を以前にダウンロードした場合は、削除して、最新バージョンに置き換えます。
    3. 管理サーバーがすでに実行されている場合は、再起動します。
  5. WebLogic Remote Consoleアプリケーションを起動します。

ブラウザ・アプリケーション

コンピュータにJava SE 11以降がインストールされている必要があります。 Javaバージョンを確認するには、コマンドラインでjava -versionと入力します。

  1. WebLogic Remote Consoleをダウンロードします。 https://github.com/oracle/weblogic-remote-console/releasesに移動して、console.zipをダウンロードします。
  2. ZIPアーカイブを任意のディレクトリに抽出します。 コンテンツが抽出されると、次を含むconsoleディレクトリが作成されます:
    • コンソール実行可能ファイルconsole.jar
    • libsサブディレクトリ内のコンソールを使用するために必要なライブラリ
    • WebLogic Serverドメインにインストールするコンソール拡張(console-rest-ext-2.0.war)
  3. オプション: WebLogic Remote Console拡張機能をWebLogic Serverドメインにインストールします。 WebLogic Remote Console拡張では、WebLogic Remote Consoleを使用するときに追加機能が追加されます。
    1. ドメイン・ホームの下にmanagement-services-extディレクトリを作成します。
    2. https://github.com/oracle/weblogic-remote-console/releasesからconsole-rest-ext-2.0.warをダウンロードし、前のステップで作成したmanagement-services-extディレクトリに保存します。 以前のバージョンの拡張機能を以前にダウンロードした場合は、削除して、最新バージョンに置き換えます。
    3. 管理サーバーがすでに実行されている場合は、再起動します。
  4. コマンド・ウィンドウを開き、コマンドラインでjava -jar <console_home>/console.jarと入力します。<console_home>はインストーラを解凍したディレクトリです。
  5. ブラウザ・ウィンドウを開き、アドレス・バーにhttp://localhost:8012と入力します。

プロバイダへの接続

WebLogic Remote Consoleは、WebLogic Server Administration Serverに接続するプロバイダと、WebLogic Deploy Tooling (WDT)モデル・ファイルおよびプロパティ・リストを編集するプロバイダをサポートしています。

  1. デスクトップ・アプリケーションまたはブラウザでWebLogic Remote Consoleを開きます。

  2. 「起動タスク」ダイアログ・ボックスからプロバイダを追加します。 または、表示されない場合は、キオスクを展開し、「⋮」をクリックします。 次のプロバイダを使用できます:

    • 管理サーバー接続プロバイダの追加(管理サーバーが実行されている必要があります)
    • WDTモデル・ファイル・プロバイダの追加
    • WDTコンポジット・モデル・ファイル・プロバイダの追加
    • プロパティ・リスト・プロバイダの追加
    • 新規WDTモデル・ファイルのプロバイダの作成
    • 新規プロパティ・リストのプロバイダの作成

    詳細は、「プロバイダ・タイプ」を参照してください。

    デスクトップ・アプリケーションでは、アプリケーションをクローズして再オープンしたときに保持されるプロジェクトにプロバイダを編成できます。 詳細は、「プロジェクト」を参照してください。

  3. プロジェクトにプロバイダを追加します。 プロジェクト名の横にある「⋮」をクリックし、プロバイダ・タイプを選択します。

  4. キオスクからプロバイダを選択します。

  5. ドメイン構成のレビューまたは編集を開始します。
