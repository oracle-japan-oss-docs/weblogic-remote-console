ドキュメントへの貢献

WebLogic Remote Consoleドキュメントは、「Hugo静的サイト・ジェネレータ」を使用して生成されます。 webページは、WebLogic Remote Consoleと同じリポジトリ内で、マークアップで作成され、ホストされます。 これにより、ドキュメントに直接編集や改善を提案しやすくなります。

ドキュメントを更新するには:

  1. リポジトリをクローニングします。

    $ git clone https://github.com/oracle/weblogic-remote-console

  2. 新しいブランチを作成します。

    $ git checkout -b <your-branch>

  3. ドキュメントを変更します。 hugo/staging/contentのマークダウン・ソース・ファイルを編集します。

  4. サイトをローカルにビルドして変更内容を表示します。

    cd hugo/staging
hugo server -b http://localhost:1313/weblogic-remote-console

    http://localhost:1313/weblogic-remote-console/のローカル・サイトを表示します。

  5. 変更を送信する準備ができたら、ブランチをオリジンにプッシュし、プル・リクエストを送信します。 「WebLogic Remote Consoleへの貢献」ドキュメントのガイドラインに従ってください。

    あなた「のみ」がhugo/staging/content領域の変更をチェックインしていることを確認します。 サイトをビルドせず、静的ファイルのコミットを試行してください。
