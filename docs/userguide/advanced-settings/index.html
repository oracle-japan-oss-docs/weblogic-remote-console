接続設定のカスタマイズ

WebLogic Remote Consoleのデフォルトの接続設定は、Javaシステム・プロパティを変更することによってカスタマイズできます。

  • デスクトップ・アプリケーションでのJavaシステム・プロパティの編集
  • ブラウザベースのアプリケーションでJavaシステム・プロパティを編集
  • Javaシステム・プロパティ

可能なカスタマイズは次のとおりです:

  • SSL/TLSを使用したWebLogicドメインへの接続
  • WebLogic Remote Consoleホストのリスニング・アドレスを指定
  • サーバーのネットワーク・タイムアウト設定の変更
  • WebLogicドメインへの接続でホスト名検証の無効化
  • webブラウザのサポートに必要な場合は、SameSite cookie属性を設定
  • kubernetesで実行されているWebLogicドメインに接続

デスクトップ・アプリケーションでのJavaシステム・プロパティの編集

WebLogic Remote Consoleデスクトップ・アプリケーションをインストールした場合は、アプリケーションを起動する前に、優先設定でconfig.jsonファイルを編集できます。

config.jsonファイルは、次の場所にあります:

  • Linux: $HOME/.config/weblogic-remote-console/config.json
  • macOS: /Users/<user>/Library/Application Support/weblogic-remote-console/config.json
  • Windows: C:\Users\<user>\AppData\Roaming\weblogic-remote-console\config.json
    • config.jsonに入力されたWindowsファイル・パスは、正しくエスケープする必要があります。 たとえば、C:\\Users\\Jane\\myTrust.jksとしてC:\Users\Jane\myTrust.jksと入力します。

config.jsonファイルを手動で作成する必要がある場合があります。

たとえば、localhost (IPアドレス127.0.0.1)以外のホストでリスニングするようにWebLogic Remote Consoleを設定するには:

config.jsonファイルで、次のように入力します:

{"server.host": "0.0.0.0"}

ブラウザベースのアプリケーションでJavaシステム・プロパティを編集

WebLogic Remote Consoleブラウザ・バージョンをインストールした場合は、WebLogic Remote Consoleを起動するたびにコマンドラインでJavaシステム・プロパティを設定します。

たとえば、localhost (IPアドレス127.0.0.1)以外のホストでリスニングするようにWebLogic Remote Consoleを設定するには:

コマンドラインで、次のコマンドを入力します。

java -Dserver.host=0.0.0.0 -jar <console_home>/console.jar

Javaシステム・プロパティ

プロパティ デフォルト config.json システム・プロパティ
console.disableHostnameVerification false console.disableHostnameVerification -Dconsole.disableHostnameVerification
console.enableSameSiteCookieValue false console.enableSameSiteCookieValue -Dconsole.enableSameSiteCookieValue
console.valueSameSiteCookie lax console.valueSameSiteCookie -Dconsole.valueSameSiteCookie
console.readTimeoutMillis 20000 console.readTimeoutMillis -Dconsole.readTimeoutMillis
console.connectTimeoutMillis 10000 console.connectTimeoutMillis -Dconsole.connectTimeoutMillis
server.host 127.0.0.1 server.host -Dserver.host
server.port 8012 server.port -Dserver.port
javax.net.ssl.trustStore <java-home>/lib/security/jssecacertsまたは<java-home>/lib/security/cacerts javax.net.ssl.trustStore -Djavax.net.ssl.trustStore
javax.net.ssl.trustStoreType jks javax.net.ssl.trustStoreType -Djavax.net.ssl.trustStoreType

SSL/TLSを使用したWebLogicドメインへの接続

WebLogicドメインへの接続ウィンドウでドメインURLにHTTPSを指定した場合、WebLogic Remote ConsoleはSSL/TLSを使用してWebLogicドメインと通信します。

SSL/TLS接続には、WebLogicドメインでの信頼が必要です。信頼構成は、基礎となるJDK JSSEサポートによって処理されます。 デフォルトでは、JDKはJDKで提供されているcacertsトラスト・ストアを使用します。 WebLogicドメインに追加の信頼性、個別の信頼性が必要な場合、またはWebLogicデモの信頼性(demotrust.jks)を使用している場合は、リモート・コンソールの起動時にJDKシステム・プロパティを使用できます。

次のいずれかのオプションを使用して、JDKでSSL/TLS信頼を構成できます:

  • keytoolコマンドを使用して、JDKで指定されたcacertsトラスト・ストアに、必要な信頼証明書をインポートします。

  • JSSEサポートのJDK Javaシステム・プロパティをコマンドラインで更新します(ブラウザのみ)。 次に例を示します。 

    java -Djavax.net.ssl.trustStore="/home/user/mytrust.jks" -Djavax.net.ssl.trustStoreType="JKS" -jar <console_home>/console.jar

    このコマンドでは、<console_home>はインストーラを解凍したディレクトリです。

  • プロパティ・ファイルでJSSEサポートのJDK Javaシステム・プロパティを更新して、トラスト・ストアのロケーションとタイプを構成します(アプリケーションのみ)。

    1. config.jsonという名前のファイルを作成し、「オペレーティング・システムの適切なロケーション」に保存します。
    2. 次のプロパティを追加します。 
      {
  "javax.net.ssl.trustStore": "/home/user/mytrust.jks",
  "javax.net.ssl.trustStoreType": "JKS"
}

      Windowsでは、パス文字\をエスケープする必要があります。次に例を示します:

      {
  "javax.net.ssl.trustStore": "C:\\Users\\user\\mytrust.jks",
  "javax.net.ssl.trustStoreType": "JKS"
}
    3. 変更を保存し、リモート・コンソール・アプリケーションを開きます。

WebLogic Remote Consoleホストのリスニング・アドレスを指定

ブラウザと同じコンピュータで実行されていないWebLogic Remote Consoleに接続するには、Javaシステム・プロパティを使用して、リモート・コンソールのホストおよびポートにデフォルト以外の値を指定できます。 このプロセスは、リモート・コンソールをconsole.zipを使用するか、デスクトップ・アプリケーションを使用するかによって異なります。

ブラウザ (console.zip)

console.zipの実行時にリスニング・アドレスを指定するには:

  1. リモート・コンソールがインストールされているホストで、コマンド・ウィンドウを開きます。

  2. コマンド行で、リモート・コンソールの起動時に次のJavaシステム・プロパティを設定します:

  • -Dserver.host=<host-address>: localhost (IPアドレス127.0.0.1)以外のホストでリモート・コンソールをリスニングします

  • リモート・コンソールを8012以外のポートにバインドするには、-Dserver.port=<port-number>を指定します。

    次に例を示します。

    java -Dserver.host=0.0.0.0 -Dserver.port=8092 -jar <console_home>/console.jar

    この例では、0.0.0.0は、ホストがそのコンピュータ上のすべてのIPアドレスでリスニングしていることを指定し、<console_home>はインストーラを解凍したディレクトリを表します。

  1. ローカル・コンピュータでブラウザ・ウィンドウを開き、次のように入力します:

    http://hostname:8092

    この例では、hostnameはリモート・コンソールを起動したコンピュータを表し、IPアドレスまたはDNS名(myhost.example.comなど)にできます。

    別のコンピュータで実行されているリモート・コンソール・プロセスに接続すると、機密データがネットワーク上で公開されるリスクがあります。

  2. WebLogicドメインに接続ウィンドウで、ドメインの管理者ユーザー名、パスワードおよびURLを入力します。

    リモート・コンソールがブラウザと同じコンピュータで実行されていない場合、リモート・コンソール・プロセスを実行しているコンピュータからWebLogicドメインURLにアクセスできる必要があります。

デスクトップ・アプリケーション

デスクトップ・アプリケーションの実行時にリスニング・アドレスを指定するには:

  1. リモート・コンソールがインストールされているホストで、「オペレーティング・システムに適用可能なロケーション」にconfig.jsonファイルを作成します。

  2. config.jsonで、次のプロパティを追加し、必要なホスト名およびポートを反映するように値を更新します。

    {
  "server.host": "0.0.0.0",
  "server.port": "8092"
}

  3. 同じコンピュータで、コマンド・ウィンドウを開き、オペレーティング・システムに適用可能なコマンドを入力します:

    • Linux: weblogic-remote-console --headless
    • macOS: /Applications/WebLogic Remote Console.app/Contents/MacOS/WebLogic Remote Console --headless
    • Windows: C:\Users\<user>\AppData\Local\Programs\weblogic-remote-console\WebLogic Remote Console.exe --headless

  4. ローカル・コンピュータでブラウザ・ウィンドウを開き、次のように入力します:

    http://hostname:8092

    この例では、hostnameはリモート・コンソールを起動したコンピュータを表し、IPアドレスまたはDNS名(myhost.example.comなど)にできます。

    別のコンピュータで実行されているリモート・コンソール・プロセスに接続すると、機密データがネットワーク上で公開されるリスクがあります。

  5. WebLogicドメインに接続ウィンドウで、ドメインの管理者ユーザー名、パスワードおよびURLを入力します。

    リモート・コンソールがブラウザと同じコンピュータで実行されていない場合、リモート・コンソール・プロセスを実行しているコンピュータからWebLogicドメインURLにアクセスできる必要があります。

サーバーのネットワーク・タイムアウト設定の変更

WebLogicドメインで使用される接続および読取りタイムアウト設定のデフォルトをリモート・コンソールから変更するには、次のJavaシステム・プロパティを変更します:

  • レスポンスの待機時にタイムアウトをconsole.readTimeoutMillis=<millis>に設定します。デフォルト: 20 seconds
  • 接続の待機時にタイムアウトをconsole.connectTimeoutMillis=<millis>に設定します。デフォルト: 10 seconds

次に例を示します。

java -Dconsole.readTimeoutMillis=60000 -Dconsole.connectTimeoutMillis=30000 -jar <console_home>/console.jar

この例では、<console_home>はインストーラを解凍したディレクトリを表し、WebLogicドメインからのレスポンスを放棄する前にコンソールが60 seconds待機します。

またはconfig.jsonに、次を追加します:

{
    "console.readTimeoutMillis": "60000",
    "console.connectTimeoutMillis": "30000"
}

ネットワーク・タイムアウト設定を変更すると、プライマリの影響はコンソール・スレッドのレスポンス時間になり、タイムアウトが発生してもブラウザにはデータが表示されません。 タイムアウトが発生する場所の例には、WebLogicがサーバーの実行時モニタリング・アクションなど、より長い初期化または実行時間を経験するリクエストがあります。

WebLogicドメインへの接続でホスト名検証の無効化

WebLogicデモの信頼を使用してWebLogicドメインに接続する場合は、ホスト名検証を無効にする必要がある場合があります。 ホスト名検証を無効にすると、リモート・コンソールは、接続が確立されたURLのホスト名がSSL接続の一部として送り返されるデジタル証明書のホスト名と一致することの確認をスキップします。

本番環境では、デモ証明書の使用やホスト名検証の無効化はお薦めしません。

ホスト名検証を無効にするには、必要に応じてconsole.disableHostnameVerificationプロパティをtrueに設定します。 デフォルトは「false」です。

たとえば、ホスト名検証が無効になっているSSL/TLSを使用してドメインに接続するには:

java -Dconsole.disableHostnameVerification=true -Djavax.net.ssl.trustStore="/<path-to-wl-home>/server/lib/DemoTrust.jks" -Djavax.net.ssl.trustStoreType="JKS" -jar <console_home>/console.jar

またはconfig.jsonファイルで、次を追加

  {
    "console.disableHostnameVerification": "true",
    "javax.net.ssl.trustStore": "/<path-to-wl-home>/server/lib/DemoTrust.jks",
    "javax.net.ssl.trustStoreType": "JKS"
  }

Windowsでは、パス文字\をエスケープする必要があります。次に例を示します:

  {
    "console.disableHostnameVerification": "true",
    "javax.net.ssl.trustStore": "C:\\<path-to-wl-home>\\server\\lib\\DemoTrust.jks",
    "javax.net.ssl.trustStoreType": "JKS"
  }

webブラウザのサポートに必要な場合は、SameSite cookie属性を設定

WebLogic Remote ConsoleがWebLogicドメインとの接続を確立すると、Webブラウザ・セッションでHTTP Cookieが確立されます。

セキュリティ上の理由から、HTTPセッションCookieを受け入れるには、WebブラウザにHTTP CookieのSameSite属性を設定する必要があります。 リモート・コンソールの動作を制御する設定は2つあります:

  • console.enableSameSiteCookieValue=trueを設定して、HTTP CookieのSameSite属性(デフォルト: false)を含めます
  • console.valueSameSiteCookie="<value>"を設定して、SameSite属性の値(デフォルト: Lax)を指定

次に例を示します。

java -Dconsole.enableSameSiteCookieValue=true -jar <console_home>/console.jar

または

{"console.enableSameSiteCookieValue": "true"}

値Laxを持つSameSite属性を含むHTTPセッションCookieになります。

kubernetesで実行されているWebLogicドメインに接続

WebLogic Remote Consoleの利点の1つは、Kubernetesで実行されているWebLogic Serverドメインに接続および管理する機能です。 Kubernetesで実行されているWebLogic Serverドメインへのアクセスを設定する方法の詳細は、「WebLogic Kubernetes Operatorユーザー・ガイド」の「リモート・コンソールの使用」を参照してください。
