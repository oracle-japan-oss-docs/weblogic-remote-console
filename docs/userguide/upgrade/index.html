WebLogic Remote Consoleのアップグレード

パッチを含む最新バージョンのWebLogic Serverで動作するように定期的に更新されるため、WebLogic Remote Consoleの最新バージョンを最新の状態に保つことをお薦めします。

WebLogic Remote Console拡張は、新しい機能を組み込むために定期的に更新されます。 WebLogic Remote Consoleをアップグレードする場合は、最新の機能をすべて使用できるように、使用可能な新しいバージョンの拡張機能があるかどうかを確認します。

拡張機能の以前のバージョンを削除して置き換えます。 拡張機能の複数のバージョンをインストールしたままにすると、予期しない動作が発生する可能性があります。

現在のバージョンはconsole-rest-ext-2.0.warです。

デスクトップ・アプリケーション

デスクトップ・アプリケーションの新しいバージョンを確認するには、「ヘルプ」 > 「更新を確認」に進みます。 新しいバージョンが使用可能な場合は、最新のWebLogic Remote ConsoleリリースのGitHub webページに移動することを選択できるダイアログ・ボックスが開きます。 すでに最新バージョンの場合は、これ以上のアクションは必要ありません。

config.jsonを使用してWebLogic Remote Consoleのデフォルト接続設定をカスタマイズする場合は、アップグレード後に簡単にリストアするために、config.jsonを別のロケーションにバックアップすることを検討してください。

  1. https://github.com/oracle/weblogic-remote-console/releasesから最新バージョンをダウンロードします。
  2. WebLogic Remote Consoleを閉じます。
  3. WebLogic Remote Consoleの新しいバージョンのインストーラを実行します。 「WebLogic Remote Consoleのインストール」の手順に従います。

Windows上で2.2.0以前からアップグレードする場合、以前のバージョンをアンインストールしようとするとアップグレードが失敗することがあります。 修正するには、PATH環境変数を編集して、%SystemRoot%\system32がfindコマンドを含む最初のエントリであることを確認してから、アップグレードを再試行します。

ブラウザ・ベースのアプリケーション

  1. https://github.com/oracle/weblogic-remote-console/releasesから最新バージョンをダウンロードします。
  2. WebLogic Remote Consoleを停止します。
  3. コンソールのホーム・ディレクトリ(console.zipからファイルを抽出した場所)を削除します。
  4. 「WebLogic Remote Consoleのインストール」のインストール手順に従います。
